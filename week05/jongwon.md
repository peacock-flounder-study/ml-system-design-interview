# 유해 콘텐츠 감지

- 두 가지 범주
	- 유해한 콘텐츠 : 폭력, 노출, 자해, 혐오 발언 등
	- 나쁜 행위, 나쁜 행위자 : 가짜 계정, 스팸, 피싱, 조직적인 비윤리적인 활동 및 기타 안전하지 않은 활동

- 탐지 대상
	- 여기서는 유해 콘텐츠 위주
- 텍스트 + 이미지 + 동영상 조합
- 다양한 언어 감지
	- 사전 학습된 다국어 모델이 있음
- 매일 5억개 이상의 새로운 게시물. 사람이 하루 10,000개 주석 달 수 있음
- 신고 시스템 있음
- 사용자에게 게시물 삭제 이유 설명해야 함
- 지연시간 -> 유해한 종류에 따라 다를 수 있음

## 머신러닝 작업으로 문제를 구조화
### 목표 정의
- 유해한 게시물 정확히 예측

### 시스템의 입력 및 출력 지정
- 게시물 입력
- 유해할 확률 출력

- 시스템은 멀티 모달 고려해야 함
- 정확히 예측하려면 모든 형태 고려해야 함
- 이런 경우 late fusion, early fusion 등이 있음
	- late fusion
		- 머신러닝 모델이 독립적으로 서로 다른 양식 처리하고, 예측 결합하여 최종 예측
		- 각 모델을 독립적으로 훈련, 평가, 개선 가능
		- 단, 개별 모델 학습하려면 개별 데이터가 필요해 시간과 비용이 들고
		- 각 양식은 무해해도 종합하면 유해할 수도 있다. 텍스트와 이미지가 결합된 밈 등
	- early fusion
		- 미리 다양한 양식을 결합한 후, 이를 기반으로 예측 수행
		- 하나의 훈련 데이터만 있으면 되고
		- 데이터의 조합으로 유해성 판단 가능
		- 그러나 양식 사이의 복잡한 관계 때문에 학습이 더 어렵다. 많은 학습 데이터가 필요하다
	- 어떤 방법을 사용?
		- 이른 융합을 사용해야 더 잘 포착할 수 있고
		- 이미 게시물은 많고, 많이 올라오니 데이터는 충분함

### 적합한 머신러닝 유형 선택
- 단일 이진 분류기(single binary classifier)
	- 융합된 피처를 입력받아 확률을 예측
	- 단, 이는 어떤 유해성 등급에 속하는지 판단이 어려움
	- 삭제 이유를 설명하기 어려움
	- 시스템이 잘 작동하지 않을 때 유해한 클래스를 모르므로 어떤 클래스에 대해 잘 작동하지 않는지 알 수 없어 시스템 개선이 어려움
- 유해 등급당 하나의 이진 분류기
	- 각 유해 등급 하나당 하나의 이진 분류기 채택
	- 융합된 피처를 입력받아 유해 등급으로 분류될 확률을 예측
	- 삭제 이유를 설명할 수 있고, 다양한 모델을 모니터링하며 개선하는 것도 가능
	- 모델이 여러 개라 학습 및 유지 관리 비용이 큼
	- 학습 비용도 큼
- 다중 라벨 분류기 (multi-label classifier)
	- 단일 모델로 다중 라벨 분류
	- 공유 모델을 사용해 모델을 훈련하고 유지하는 비용이 절감됨
	- WPIE(Whole Post Integrity Embeddings)
		- [Harmful content can evolve quickly. Our new AI system adapts to tackle it.](https://ai.meta.com/blog/harmful-content-can-evolve-quickly-our-new-ai-system-adapts-to-tackle-it/)
	- 그러나 공유 모델을 사용하면 각 유해 등급의 확률을 예측하려면 입력 피처를 다르게 변환해야 할 수 있으므로 이상적이지 않음
		- 유해 카테고리마다 봐야하는 특징이 다른데 이걸 공통 표현으로 처리하려고 하면 어떤 카테고리에서는 성능이 낮아질 수도 있음
		- WPIE도 전체 성능은 올라갔지만 일부 카테고리에서는 떨어짐
			- 그래서 아래서는 일부는 공유하되 태스크 출력 층을 따로 둠
		- -> 찾아봤는데 잘 모르겠습니다..  결국 위에서 설명한 여러 성질을 합쳐서 한 번에 본다 같음
- 다중 작업 분류기 (multi-task classifier)
	- 모델이 여러 작업을 동시에 학습
	- 작업 간의 유사성을 학습할 수 있고, 특정 입력 변환이 여러 작업에 도움이 될 때 불필요한 계산을 피할 수 있다
	- 폭력, 노출 등 다양한 종류의 유해 콘텐츠를 서로 다른 작업으로 취급하고, 다중 작업 분류 모델을 통해 각 작업을 학습한다
	- 작업이 공유 계층, 작업별 계층으로 나뉨
		- 공유 계층
			- 입력 피처를 새로운 피처로 변환하는 은닉 계층 집합
		- 작업별 게층
			- 독립적인 머신러닝 출력 계층 (classification head)
	- 세 가지 장점
		- 단일 모델이라 훈련 및 유지 관리 비용 적음
		- 공유 계층 덕에 중복 계산 방지, 다중 작업 분류 효율적으로 수행
		- 각 학습 데이터는 다른 작업 학습에 도움을 준다 (공유 계층 덕)
## 데이터 준비
- 사용 데이터
	- 사용자
	- 게시물
	- 사용자와 게시물 간 상호작용
		- 좋아요, 댓글, 저장, 공유 등
		- 유해하다고 신고 데이터

### 피처 엔지니어링
- 게시물은 다음 요소
	- 텍스트 콘텐츠
		- 전처리, 벡터화 등 필요
		- 벡터화에는 통계적, 머신러닝 기반 방법 사용 가능
		- BoW, TF-IDF 등의 통계적 방법은 구현이 쉽고 계산 속도도 빠름
		- 그러나 텍스트 의미 인코딩 불가
		- 콘텐츠 의미 이해가 중요하므로 머신러닝 기반 방법 사용
		- 텍스트를 피처 벡터로 변환하기 위해 사전 학습된 트랜스포머 모델 사용
		- 기존 BERT에는 두 가지 문제
			- 모델이 커 오래 걸림. 온라인 예측에는 적합하지 않음
			- 영어 전용 데이터로만 학습되었음
		- DistilBERT 위 두 가지 문제 해결
			- Distillation로 만든 BERT 모델
			- 훨씬 효율적 
				- BERT에 비해 40%나 가벼워졌고 60%나 빨라졌으며 97% 성능을 유지
	- 이미지, 영상
		- 이런 비정형 데이터 준비할 때는 일반적으로 다음 두 단계
		- 전처리
			- 데이터 디코딩, 크기 조정, 정규화
		- 피처 추출
			- 전처리 후 사전 학습된 모델로 비정형 데이터를 피처 벡터로 변환
			- 이미지는 CLIP의 비주얼 인코더, SimCLR 등 사전 학습된 모델
			- 동영상은 VideMoCo 같은 모델
	- 게시물에 대한 사용자 반응
		- 콘텐츠가 애매하면 사용자 반응이 효과적일 수도 있음
		- 유용한 피처
			- 좋아요, 공유, 댓글 및 신고 수 
				- 이런 수치 값을 조정하여 모델 학습 중 수렴 속도를 높일 수 있다
			- 댓글
				- 사전 학습된 모델로 댓글의 임베딩 생성
				- 임베딩 집계하여 최종 임베딩 얻음
	- 작성자 피처
		- 유해성 판단을 위해 작성자의 과거 상호작용 사용 가능
		- 작성자 위반 이력
			- 위반 횟수, 사용자 신고 총수, 비속어 단어 비율 등
		- 작성자 개인 정보
			- 나이, 성별, 도시와 국가 등
		- 작성자 계정 정보
			- 팔로워 및 팔로잉 수, 계정 나이
	- 컨텍스트 정보
		- 시간, 단말 등

## 모델 개발
### 모델 선택
 - 다중 작업 학습에 가장 맣이 사용되는 모델은 신경망
### 모델 훈련
#### 데이터세트 구성
- 모델 입력(피처), 출력(라벨)로 구성
- 입력은 오프라인에서 게시물 일괄 처리하고 융합된 피처 계산
- 입력에 대한 라벨은 아래 정도 가능
	- 수동 라벨링, 자동 라벨링
- 평가 데이터는 수동 라벨링으로 정확성을 올리고 자동 라벨링은 라벨링 속도에 우선순위를 둔다

#### 손실 함수 선택
- 멀티태스크 신경망 훈련은 일반적인 신경망 훈련과 아주 유사
- 순전파는 예측을 위한 계산 수행
- 손실 함수는 예측의 정확성 측정
- 역전파는 다음 반복에서 손실을 줄이기 위해 모델 매개변수 최적화

- 여기서는 각 작업이 이진 분류이므로 크로스 엔트로피 등의 표준 이진 분류 손실 채택
- 전체 손실은 각 작업 손실을 결합하여 계산

- 멀티 모달 시스템 학습할 때 흔히 발생하는 문제는 과대적합이다
- 학습 속도가 여러 모달리티에 따라 다른 경우 한 모달리티가 학습 과정의 대부분을 차지할 수도 있다
- 이런 경우 Gradient Blending, Focal Loss 등을 사용한다.
	- Gradient Blending
		- 각 모달리티의 Gradient를 적절히 섞어서 한 모달리티로 쏠리지 않도록 하는 것
		- 모달리티별 손실을 따로 계산
		- 모달리티별 gradient 크기나 유용성을 측정
		- 더 중요한 쪽, 덜 학습된 쪽, 덜 반영된 쪽의 비중을 조정
		- 최종적으로 섞인 gradient로 업데이트
	- Focal Loss
		- 어려운 샘플에 더 집중
		- 쉬운 문제의 가중치를 낮추고, 어려운 문제에 가중치를 높임

## 평가
### 오프라인 지표
- 이진 분류 모델은 precision, recall, F1-Score 등을 사용
- 그러나 precision, recall 만으로는 전반적인 성능 파악이 어려움
- 이때문에 PR(precision-recall) 곡선과 ROC(Receiver Operating Characteristic) 곡선을 사용해 이런 한계를 해결

#### PR 곡선
- 정밀도, 정확도 상충 관계를 보여준다
- 이를 요약하기 위해 PR-AUC(PR 곡선 아래 면적)을 계산함
- 높을수록 모델이 정확

### 온라인 지표
- 유행성(prevalaence)
	- 플랫폼의 모든 게시물 중에서 차단하지 않은 유해한 게시물 비율
	- 이 지표의 단점은 모든 유해한 게시물을 동등하게 취급한다는 것
		- 조회수 높은 게 더 나쁜 영향을 줌
- 유해한 노출(harmful impression)
	- 유행성보다 선호
	- 유해한 노출 수로 실제로 끼친 피해를 반영할 수 있음
- 유효한 이의 제기 건수
	- 유해하다고 판단되었으나 이의 신청을 통해 취소된 게시물 비율
- 사전 예방 비율
	- 사용자가 신고하기 전에 시스템에서 삭제한 유해 게시물 비율
- 유해 등급 별 사용자 신고 수
	- 각 유해 등급별 사용자 신고 수를 조사해 시스템 성능 측정

## 서빙
### 유해 콘텐츠 감지 서비스
- 새 게시물 업로드되면 유해 가능성 예측

### 위반 단속 서비스
- 높은 신뢰도로 유해성 예측한 경우 즉시 삭제하고 삭제 이유를 사용자에게 알림

### 강등 서비스
- 낮은 신뢰도로 유해성 예측하는 경우 확산 가능성을 줄이기 위해 일시적으로 게시물 등급을 내림
- 그리고 사람이 수동 검토 후 라벨 할당 or 등급 복구
