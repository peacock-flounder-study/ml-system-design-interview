# 구글 스트리트 뷰 블러링 시스템

- 객체 위치 예측 : 회귀 문제
- 바운딩 박스 클래스 예측 : 다중 클래스 분류 문제

### 2단계 네트워크

두 개의 개별 모델 사용

Region Proposal Network(RPN): 이미지를 스캔하여 객체가 될 가능성의 후보 영역 제안
Classifier : 객체 클래스로 분류

R-CNN, Fast R-CNN, Faster-RCNN

### 1단계 네트워크

두 단계를 결합. YOLO, SSD

2단계는 일반적으로 느리지만 더 정확하다. 훈련 데이터가 증가하거나 더 빨리 예측해야 하는 경우 1단계 네트워크로 전환한다.

(YOLO는 이전에 anchor-based 방식이어서, 위치마다 여러 개의 미리 정의된 anchor를 깔아놓고 학습했음. v8부터 각 픽셀이 직접 bbox를 예측하는 형태.)
(Anchor-based는 작은 객체가 많거나 정밀한 박스 튜닝이 필요하면 유리하긴 함. 그러나 더 단순하고 빠른 anchor-free가 선호되는 추세)

## 데이터 증강

- 온라인 데이터 증강 : 추가 스토리지가 불필요, 훈련 속도 느림
- 오프라인 데이터 증강 : 추가 스토리지가 필요, 훈련 속도 빠름

## 모델

**feature map** : 입력 이미지와 필터(커널) 간의 컨볼루션 연산의 출력. 특정 기능이나 패턴의 존재를 나타내는 결과 값의 matrix

순전파, 손실 계산, 역전파

객체 감지 모델은 1/ 예측된 객체의 bounding box가 ground truth랑 최대한 겹쳐야 함 2/ 객체 클래스에 대한 예측 확률이 정확해야 함

## 평가

어떨 때 예측이 맞다고 할 수 있을까?

IOU(Intersction Over Union) : 두 바운딩 박스 사이의 중첩을 측정한다. IOU 임계값을 통해 올바른 감지인지 판단한다.

### 오프라인 지표

- precision
- average precision(AP)
- mean average precision(mAP)

Threshold에 의존해서 모델을 평가하는 것이 아니라, 여러 Threshold에서 precision-recall 관계를 보기 위해 AP가 필요.
mAP는 모델의 전체 점수

MS에서 만든 COCO 벤치마크를 활용함

## 서빙

추론하는 동안 바운딩 박스를 객체당 하나의 바운딩 박스로 좁히는 것이 중요하다.

**NMS(non-maximum suppression)** : 가장 적합한 바운딩 박스를 선택하도록 설계된 후처리 알고리즘

구현이 간단하고, 매우 빠름(real-time detection에서 주로 활용)

단순 IOU 대신 중심 거리까지 고려하는 NMS를 사용하기도 함. DETR 계열은 NMS을 사용하지 않기도 함.


블러링은 GPU를 쓴다
=> 박스 블러(속도 빠름, 상대적으로 텍스쳐 뭉개짐), 가우시안 블러(상대적으로 속도 느림, 텍스쳐 정보 적절히 살아남) 등이 있음.
