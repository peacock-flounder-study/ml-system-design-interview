##  4장. 유투브 동영상 검색

사용자가 입력한 텍스트 쿼리를  받아 관련도에 따라 정렬, 순위가 매겨진 동영상 목록을 출력해야한다.

- input : 텍스트 쿼리
- output : 동영상 목록
- 조건
    - 영어 지원
    - 개인화가 필요없음
    - 영상은 10억개 이상

#### 머신러닝 유형 선택하기

- 시각 검색
    - 표현학습 : 시각적 콘텐츠 처리해서 동영상을 검색하는 데 사용
    - 텍스트, 동영상을 각각 인코딩해서 동영상과 텍스트 간의 유사도 점수를 스칼라 곱으로 계산
- 텍스트 검색
    - 역색인 구조로 머신러닝 기반은 아니며, 텍스트 검색어와 가장 유사한 제목이나 태그를 조회할 수 있다.
    - elasticsearch

### 데이터 준비

- feature 엔지니어링
    - 텍스트, 동영상과 같은 비정형 데이터는 수치로 변환이 필요하다.
    - 텍스트
        - 정규화로 단어의 문장과 일관성 유지하기 (dog,dogs,DOG!  ⇒  dog)
        - 토크나이저를 통해 텍스트 조각인 토큰 단위로 분해한다.
            - ,와 같은 특정 구분 기호에 따라 나누는  단어 토큰화, 텍스트를 하위 단어로 분할하는 서브워드 토큰화, 문자 집합으로 분할하는 문자 토큰화 알고리즘 등이 있음
        - 토큰을 확보하고 나면 숫자 ID 값으로 변환하여 임베딩한다.
            - 해싱(피쳐 해싱이나 해싱 트릭)을 통해 룩업 테이블을 유지하지 않고도 메모리를 효율적으로 사용하여 ID를 가져오는 방법을 주로 사용
    - 동영상 데이터도 프레임 단위로 디코딩해서 샘플링, 스케일링 등을 통해서 전처리를 진행

### 모델 개발하기

1. **텍스트 인코더**

텍스트를 벡터 표현으로 변환하여 두 문장의 의미가 비슷하면 임베딩이 더 유사해지는 구조

1. 통계적 방법
    - 문장을 피처 벡터로 변환하기 위해 `BoW`나 `TF-IDF`와 같은 통계 기법을 사용
        - BoW : 문장을 fixed-length 벡터로 변환해서 행렬을 생성한다.
        - TF-IDF : 단어가 컬렉션이나 corpus(말뭉치)에서 얼마나 중요한지 반영하기 위한 수치 통계이다. 동일하게 문장-단어 행렬을 생성하나 단어의 빈도에 따라 정규화한다는 차이점이 있음
2. ml 기반 기법

문장을 의미있는 단어 임베딩으로 변환하고, 두 임베딩 사이의 거리를 해당 단어들의 유사성으로 반영

- 임베딩(lookup)계층
    - 각 ID를 임베딩 벡터에 매핑한다.
    - ID와 같은 희소 피처를 고정된 크기 임베딩으로 변환할 수 있다는 장점
- word2vec
    - 단어 임베딩을 생성하는데 사용하는 모델로, shallow neural network 아키텍처를 사용해서 단어 동시출현(co-occurrence)를 활용하여 학습
    - `주변 단어로부터 중심 단어`를 예측하는 방법을 학습하기 때문에 훈련이 끝나면 의미있는 임베딩으로 변환이 가능
        - CBOW, Skip-gram이 워드투벡터 기반의 모델
- 트랜스포머 기반 아키텍처
    - 단어를 임베딩으로 변환할 때 문장에 포함된 단어의 `컨텍스트`를 고려
    - 문맥을 이해하고 의미 있는 임베딩을 생성하는데 word2vec보다 적합함 (왜?)

1. **동영상 인코더** 
2. 동영상 수준 모델
    - 전체 동영상을 처리해서 임베딩을 생성
    - 일반적으로 3차원 합성곱 또는 트랜스포머 기반의 아키텍처이며, 전체 동영상을 처리해서 계산 비용이 많이 듦
3. 프레임 수준 모델
    - 동영상 샘플 프레임을 전처리해서
    - 샘플링 프레임에 모델을 실행하여 프레임 임베딩 생성
    - 프레임 임베딩을 집계하여 동영상 임베딩 생성
    - 프레임 수준에서 동작해서 계산 비용도 적고 빠르지만, 시간에 따른 동작이나 행동 변화를 이해할 수 없으므로 이런 특성의 영상에만 사용함

1. **모델 훈련하기**
- 텍스트와 동영상 인코더를 대조학습 방식을 통해 훈련함
- 손실 계산 사진

### 평가하기

<오프라인 지표>

- precision@k나 mAp : 텍스트 쿼리는 동영상 하나와 매칭되므로 분자는 최대 1이어서 값이 낮음 (최상위로 순위를 두어도 0.1)
- recall@k : 상위 K개 관련 동영상 수/관련 동영상 전체 개수
    - 관련 동영상의 총 개수는 항상 1이기 때문에 1과 0으로 구분할 수 있음
    - 주어진 텍스트 쿼리에 대한 영상 찾는 모델 평가로는 효과적이나
    - ‘k’값에 따라 결과가 다를 수 있음. 예를 들면 모델 품질과 상관없이 관련동영상 목록에 있는 k개의 동영상에 포함 안되면 recall이 0값이어서 품질을 구분하기 어렵다.
- MRR
