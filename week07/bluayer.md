## 머신러닝

- LTR(Learning To Rank) : 지도 학습을 적용해 순위 문제를 해결.
  - pointwise : 항목 + 쿼리 -> 연관성 점수
  - pairwise : 항목 x, y + 쿼리 -> 무엇이 더 관련성 높은지
  - listwise : N 항목 리스트 + 쿼리 -> 최적의 순서

## 피처 엔지니어링

이벤트 기반 추천은 기존 추천보다 더 까다로움. 수명이 짧고 콜드 스타트가 일어나기 쉬움
피처 엔지니어링에 더 큰 노력이 필요.
- 위치 관련 피처
- 시간 관련 피처
- 소셜 관련 피처
- 사용자 관련 피처
- 이벤트 관련 피처
=> 그냥 이벤트 호스트가 고민하는 거랑 똑같음.

## 모델 개발
- 로지스틱 회귀 : 여러 입력 피처들을 가지고 이진 결과 확률 모델링. 두 개 이상의 피처가 높은 상관관계를 가질 때 학습이 잘 안됨(multicollinearity)
- 의사 결정 트리 : 데이터의 작은 변화에도 민감. 따라서 bagging - 랜덤 포레스트 (과대 적합 영향 감소), boosting - 여러 약한 분류기를 순차적 학습 & 과소적합 영향 감소 사용
- GBDT - XGBoost가 GBDT의 변형. 이미지, 동영상, 오디오 같은 비정형 데이터에서는 잘 작동하지 않음. 지속적 학습에 부적합
