## ML 스터디 1강

머신러닝 시스템 면접은 주제에 대한 이해도, 시스템 end-to-end 설계 능력, 선택의 장단점을 따져서 사고하는 과정을 통해 설계를 진행한다. 하나의 정답은 없지만, 아래 프레임워크는 생각을 구조화하는 데 도움이 되며 상황에 맞춰 적용해보자.

(디테일한 구현은 다르지만 서버 개발에서 요구하는 능력도 유사하다.)

**1. 요구사항 명확화**
- “이벤트 추천 시스템을 설계하라”와 같은 요구사항을 정확하게 파악할 수 있어야한다. 다음과 같은 질문들을 던져서 이해해보자.
- 달성해야하는 비즈니스 목표 
- 예약 수와 수익 늘리기
- 시스템이 제공해야하는 피처(데이터 특징이나 속성)
- 데이터 소스나 세트, 라벨링 여부
- 성능이나 시스템 규모
- 컴퓨팅 자원, 배포 환경같은 제약사항 등

**2.머신러닝 작업으로 문제를 구조화**
현실에서 풀어야하는 비즈니스 관점의 문제가 있고, 머신러닝 시스템에서 달성해야하는 목표는 별도로 설정해야한다.
- 머신러닝 목표 정의
<img width="434" height="159" alt="image" src="https://github.com/user-attachments/assets/ac959e1a-7241-4aa3-bf1f-e58ce09b1b06" />

- 시스템 input/output 지정
  - 산출물 목표 잡기에 가까움
  - 게시물 > ml 감지 > 유해물 여부 
- 적합한 머신러닝 유형 선택하기
<img width="480" height="233" alt="image" src="https://github.com/user-attachments/assets/a985cf11-e6a3-4f32-a87f-182878bc2e8b" />


- 머신러닝 모델은 학습 데이터를 사용할 때 특정 작업을 더 잘 학습하기 때문에 지도 학습이 더 많이 사용된다.
- 지도 학습 
    - 회귀 : 연속적인 숫자 값 예측 (집의 기대가치 예측하기)
    - 분류 : 불연속 클래스 라벨을 예측 (이미지를 개, 고양이, 토끼로 나누기)
        이진 분류: 이미지에 개가 포함되어있는가? 0과 1
        다중 클래스 분류 : 이미지를 개,고양이 / 토끼로 나눌 수 있는가?
- 비지도 학습
  - 정답이 없는 데이터를 처리하여 예측하여 데이터 사이에 의미있는 패턴을 식별
- 강화학습 
  - agents 가 시행착오를 반복해서 수행하는 방법을 찾아내는 학습법(알파고나 로봇)

**3. 데이터 준비**

머신러닝 모델은 데이터를 통해 직접 학습하기 때문에 데이터를 준비하는 작업은 필수적이다.
raw한 데이터 소스를 어떻게 가공해서 준비된 피처로 만들 것인가? 
  <img width="501" height="118" alt="image" src="https://github.com/user-attachments/assets/e411032e-71b8-4858-92f5-525c1edebc32" />

  

- 데이터 엔지니어링
    - 데이터 수집, 저장, 검색 및 처리를 위한 파이프라인 설계
    - 수집할만한 데이터 소스를 정리하고, 다양한 종류의 DB에 데이터를 저장한다.
    - ETL(Extract, Transform, Load) 로 추출, 변환,적재를 진행한다.
    - 데이터 유형은 정형/비정형으로 나뉜다.
      -정형 데이터 중에서도 데이터 유형에 따라 수치형과 범주형으로 나뉜다.
      <img width="421" height="212" alt="image" src="https://github.com/user-attachments/assets/a9bdc5b2-5f6e-4e85-b689-7498ce162ed5" />

      - 수치형 : 숫자로 표시되는 모든 데이터로, 연속성과 고유한 값을 취하는 이산형으로 나뉜다.
      - 범주형 : 이름이나 라벨로 식별할 수 있는 데이터로, 성별처럼 순서가 없는 명목형 데이터와 “좋다, 싫다“와 같이 순서가 있는 순위형 데이터가 있다.
      - 피처 엔지니어링

머신러닝 모델 개발 시에는 해당 도메인에서 가장 가치있는 피처를 선택하는 것이 중요하다. 
- 피처 중 누락된 데이터는 값을 삭제하거나 default , 평균 값 등으로 대체한다.
  - 피처 스케일링
  - 피처를 표준 범위와 분포를 갖도록 스케일링하는 프로세스
- 왜 필요한가?
  - 데이터 세트의 피처가 서로 다른 범위에 있으면 학습에 어려움이 있어 정규화나 표준화, 로그 스케일링을 거친다.
  - 특정 피처가 단순하게 숫자 범위가 넓다는 이유로 가중치가 크게 부여되어 결과에 영향을 미치는 문제를 방지하기 위함
- 이산화(bucketing)
- <img width="267" height="271" alt="image" src="https://github.com/user-attachments/assets/582fdf19-2199-42d4-9a37-ea65527bff67" />

  - 연속성 피처를 범주형 피처로 변환하는 프로세스.
  - 사용자의 키를 연속성 피처로 표현하는 대신, 버킷 단위로 나누면 모델의 목적에 따라 몇 가지 범주만 학습할 수 있도록 집중시킬 수 있다.
  - 범주형 피처를 다시 숫자로 인코딩하는 방식은 범주에 고유한 정수값을 할당하는 정수 인코딩
  - 고유 값에 새로운 이진 피처를 할당하는 원-핫 인코딩
  - 범주형 피처를 N차원 벡터에 매핑해서 사용하는 인베딩 학습이 있다.


**4. 모델 개발**
간단한 기준선을 결정하고 이 모델로 실험한 다음 더 복잡한 모델로 전환하거나 여러 모델의 조합을 사용해서 예측 품질을 높이는 프로세스
각 머신러닝 알고리즘 별로 장단점을 이해하고, 요구사항과 제약조건에 따라 적절한 모델을 선택할 수 있어야한다.

**5.모델 훈련**
모델을 선택하고나서는 훈련을 하면 된다. 
- 데이터 세트를 구성하고
- 손실함수를 선택한 다음
- 클래스 간의 데이터 불균형을 대응하기 위해 소수 클래스의 데이터 포인트에 가중치를 더 주는 방법
- 모델이 내놓는 예상 결과의 정확도를 측정하는 지표가 되기 때문에 적합한 손실함수를 선택해야함

- 처음부터 훈련하기 vs fine-tuning
- 분산 훈련
모델과 데이터 세트 크기의 증가로 여러 워커 노드에 작업을 분할하여 모델을 병렬로 훈련하는 방법
→ 훈련 시간, 데이터 양, 추론 처리시간, 배포, 성능, 메모리 등을 더 고려해야함

**7. 압축 및 배포**

- 압축
  - knowledge disillation: 작은 모델이 큰 모델을 모방하도록 훈련하기
  - pruning (가지치기) : 유용성이 낮은 매개변수를 찾아서 0으로 설정하는 프로세스
  - quantization : 보통 모델 파라미터가 사용하는 32비트 부동소수점 표현보다 더 적은 비트를 사용해서 모델 사이즈를 줄이는 방법
- 배포
  - 섀도 배포, A/B 테스트, 카나리, 인터리빙 실험, bandits 등이 있다.
<img width="432" height="464" alt="image" src="https://github.com/user-attachments/assets/dbea453c-eb64-4a62-88f7-1406178fe9a0" />


운영 환경에서 요청을 처리하려면 예측 파이프라인이 필요하다.
- 배치 예측 : 주기적으로 계산을 수행해서 미리 계산된 예측을 리턴
- 온라인 예측 : 요청이 도착하는 즉시 예측을 만들어서 반환

**8. 모니터링**

운영 환경에서 시스템 장애가 발생하는 경우를 감지하고, 모니터링 대상을 선정하여 평가 지표를 추적, 측정 및 기록해야한다.

- 학습 데이터와 실제 업무 데이터가 달라서 데이터 분포 변화가 발생할 수 있다.
- 그래서 대규모 데이터 세트를 학습하거나, 지속적으로 재학습 시켜야한다.
