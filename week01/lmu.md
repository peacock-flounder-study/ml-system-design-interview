1장 소개 및 개요
- 기본 프레임워크
	- 요구사항 명확화
		- 비지니스 목표, 시스템이 제공해야하는 피쳐, 데이터, 제약, 시스템 규모, 성능
	- 머신러닝 작업으로 문제를 구조화
		- 머신러닝 목표 정의
		- 시스템의 입력 및 출력 지정
		- 적합한 머신러닝 유형 선택
			- 지도 학습: 회귀 모델, 분류모델(이진 vs 다중)
			- 비지도 학습
			- 강화학습
	- 데이터 준비
		- 데이터 엔지니어링
			- 데이터 소스, 데이터 저장소, ETL, 데이터 유형
		- 피쳐 엔지니어링
			- 도메인 지식을 사용하여 원시 데이터에서 예상 피쳐를 선택하고 추출
			- 예측한 피쳐를 모델에서 사용할 수 있는 형식으로 변환
			- 예시
				- 누락 데이터 처리: 삭제, 대체
				- 피쳐 스테일링: 정규화, 표준화, 로그 스케일링
				- 이산화 (bucketing)
				- 인코딩
	- 모델 개발
		- 모델 선택
			- logistic regression, linear regression, decision tree, gradient boosted decision trees, random forest, support vector machine, naive bayes, FM(Factorization Machines), Neural Network
			- 고려 요소
				- 모델이 학습해야하는 데이터의 양
				- 훈련 속도
				- 선택할 hyper parameter 와 hyper parameter tuning 기법
				- 학습 지속 가능성
				- 컴퓨팅 요구사항
				- 모델의 해석 가능성
		- 모델 훈련
			- 데이터 세트 구성, 손실함수 선택, 처음부터 훈련하기와 미세조정, 분산훈련
	- 평가
		- 오프라인 평가
			- 분류: precision, recall, F1 score, accuracy, ROC-AUC, PR-AUC, confusion matrix
				- 모르는거: F1 score, ROC-AUC, PR-AUC
		- 온라인 평가
			- 지표: e.g. 클릭률, 수익 증가
	- 배포 및 서비스 제공
		- 클라우드와 온디바이스 배포 비교
	- 모델 압축
		- 지식 증류, 가지치기, 양자화
	- 운영 환경에서 테스트
		- shadow 배포
		- A/B 테스트
	- 예측 파이프라인
		- 배치 예측
		- 온라인 예측
	- 모니터링 및 인프라
		- 운영 환경에서 시스템에 장애가 발생하는 이유
			- 대규모 데이터세트를 학습한다.
			- 새로운 분포의 라벨이 지정된 데이터를 사용하여 모델을 정기적으로 재학습한다.
		- 모니터링 대상
			- 운영 관련 지표
			- 머신러닝 관련 지표
		- 인프라스트럭쳐

모의 면접 질문
- 전통적인 벡엔드 엔지니어에서 머신러닝에 대해서 얼마나 이해해야한다고 생각하시나요?
- 전통적 벡엔드 엔지니어에서 머신러닝 백엔드로 전환하기 위해서 어떤점들을 주로 공부하시나요?
- 최근 AI가 발전함에 따라서, LLM 의 중요성이 대두되고 있는데, 전통적인 머신러닝 모델이 필요할까요? 필요하다면 어떤점 때문에 필요할까요?
- 사용자 별 광고 시스템을 설계하려고 합니다. 어떤 점들을 위주로 설계할거 같을까요?
	- 연계 질문
		- 규모에 대한 질문이 들어올 경우 -> 현재 목표는 DAU 1천만
		- 정확도에 대해 질문이 들어올경우 -> 완벽하게 정확하지는 않아도 된다. 결국에는 광고 시스템이다보니 클릭율과 최종 구매율이 중요하다.
	- 백엔드 엔지니어가 이러한 광고에 대한 스키마를 설계하려고 한다.
		- 광고 추천 시스템 모델은 이미 잘 학습되어 있다고 가정하에, 해당 모델은 다음과 같은 입력을 받는다.
			- 사용자의 메타데이터 (연령, 성별)
			- 사용자의 최근 추적 데이터 (우리 사이트에서 클릭한 페이지 및 등)
		- 우리 광고 추천 시스템에서 유입되었다는 사실을 구매 사이트에 전달하고, 그게 최종적으로 구매로 이어졌다는 사실을 추적하려고 한다. 어떻게 플로우와 스키마를 설계할것인가?
